IA Simbólica

Representar conhecimento explicitamente usando símbolos.

Exemplo:

SE paciente tem febre E tosse

ENTÃO suspeita de gripe

Base de muitos sistemas especialistas clássicos.

Por que Regras?

  • Interpretabilidade
  • Explicabilidade
  • Facilidade de manutenção
  • Ideal para domínios estruturados

Estrutura de uma Regra


SE condição
ENTÃO conclusão

SE animal possui penas

ENTÃO animal é ave

Componentes

  • Base de Conhecimento
  • Memória de Trabalho
  • Motor de Inferência

Forward Chaining

Definição

Raciocínio orientado pelos dados.

Começa com fatos conhecidos e deriva novas conclusões.

Fluxo

  1. Encontrar regras aplicáveis
  2. Executar regra
  3. Adicionar novo fato
  4. Repetir até estabilizar

Exemplo


R1: SE A ENTÃO B
R2: SE B ENTÃO C

Fato inicial: A

Resultado: B → C

Passo a Passo

  1. A é verdadeiro
  2. R1 dispara → B
  3. R2 dispara → C

Vantagens

  • Excelente para monitoramento
  • Descobre consequências automaticamente
  • Ideal para sistemas reativos

Desvantagens

  • Pode gerar fatos irrelevantes
  • Explosão combinatória

Backward Chaining

Definição

Raciocínio orientado por objetivos.

Parte da hipótese e busca evidências.

Fluxo

  1. Definir objetivo
  2. Buscar regras que o produzem
  3. Verificar pré-condições
  4. Recursivamente provar subobjetivos

Exemplo


Objetivo: C

R2: B → C
R1: A → B

Fato: A

Árvore de Objetivos


C
└── B
    └── A

Quando Usar?

  • Diagnóstico médico
  • Consultas específicas
  • Sistemas especialistas

Forward vs Backward

Critério Forward Backward
Direção Dados → Objetivo Objetivo → Dados
Uso Exploração Diagnóstico

Busca em Espaço de Estados

Definição

Encontrar uma sequência de ações que leva do estado inicial ao objetivo.

Componentes

  • Estado inicial
  • Operadores
  • Teste de objetivo
  • Custo

Exemplo: Labirinto

Encontrar caminho entre início e saída.

Busca em Largura (BFS)

Características

  • Explora nível por nível
  • Completa
  • Ótima para custos uniformes

Pseudocódigo


fila = [inicial]

enquanto fila não vazia:
    n = remover_inicio()
    se objetivo(n): retornar
    adicionar filhos ao final

Busca em Profundidade (DFS)

Características

  • Explora profundamente
  • Baixa memória
  • Pode ficar presa

Pseudocódigo


pilha = [inicial]

enquanto pilha não vazia:
    n = remover_topo()
    se objetivo(n): retornar
    empilhar filhos

Busca Gulosa

Heurística

Escolhe o nó aparentemente mais promissor.

f(n) = h(n)

A*

Função de Avaliação

f(n) = g(n) + h(n)

  • g(n): custo acumulado
  • h(n): estimativa até o objetivo

Propriedades do A*

  • Completo
  • Ótimo
  • Amplamente utilizado

Comparação

Algoritmo Ótimo Memória
BFS Sim Alta
DFS Não Baixa
A* Sim Média/Alta
