Diferentes Tipos de Aprendizado

Supervisionado

  • os pesos são ajustados minimizando o erro entre a saída esperada e a saída estimada no conjunto de dados de treinamento.

Não Supervisionado

  • não utiliza saídas rotuladas. Os pesos são otimizados com base em critérios gerais, como similaridade, densidade, recompensa ou estrutura dos dados.

Causalidade vs. Correlação

"Só porque A e B acontecem juntos, não significa que A causa B."
  • Pessoas que bebem refrigerante diet costumam estar acima do peso.
  • O classificador pode dizer: "Pare de beber diet para emagrecer".
  • Erro: A condição (obesidade) causou a escolha do refrigerante, não o contrário.

Outras considerações

  • O que realmente importa no dado?
  • Nem todo atributo ajuda na decisão

Aprendizado supervisionado

Princípio

Como classificar elementos?
  • Dado um conjunto de N exemplos {(xi, yi) ∈ 𝒳 × 𝒴, i = 1..N}, de treinamento, busca-se uma função de previsão tal que y = f(x).
  • O aprendizado é supervisionado, pois os rótulos são conhecidos
  • Exemplos
    • Classificação de imagens
    • Previsões: temperatura, clima, ações...

Regressão e classificação

Aprendizado supervisionado

  • Regressão
    • Nós falamos de regressão quando 𝒴 é um sub-conjunto de \(\mathbb{R}^d\)
    • Frequentemente, avaliamos os modelos ao calcular o erro quadrático: \( (y-f(x))^2 \)
  • Classificação
    • O espaço para predição 𝒴 é um conjunto discreto
    • Classificação binária card(𝒴)= 2
    • Classificação multiclasse card(𝒴)> 2
    • Frequentemente, avaliamos os modelos ao calcular a acurácia

Função de perda (loss)

  • Estimar o erro da predição f(x) em relação aos dados reais y
    • Por convenção: \( L(y,f(x)) = \begin{cases} 0 & \text{if } y = f(x)\\ > 0 & \text{if } y \neq 0 \end{cases} \)
    • Erro da predição esperado\(R_{true}(f) = \int_{𝒳𝗑𝒴} L(y,f(x))p(x,y)dxdy \)
    • Objetivo: achar a função que minimiza o erro \(f* = \text{arg min}_{f \in \mathbb{H}} R_{true}(f) \)

Erro

  • O erro real é impossível de ser calculado porque não sabemos a distribuição real dos dados
  • Usar o erro empírico, que estima a loss em um dataset de treinamento:
    • \[ R_{emp} = \frac{1}{N} \Sigma_i^n L(y_i,f(x_i)) \]

Algumas funções de loss

  • Erro 0-1 (mais adequado para classificação)
    • \[ L(Y,f(X)) = \begin{cases} 0 & \text{if } y = f(x)\\ 1 & \text{otherwise} \end{cases} \]
  • Erro quadrático (mais adequado para regressão)
    • \( L(Y,f(X)) = (Y-f(X))^2 \)
  • Erro l1 (erro absoluto, mais adequado para regressão)
    • \( L(Y,f(X)) = | Y-f(X) | \)
  • Erro de Hinge (mais adequado para classificação)
    • \( L(Y,f(X)) = max(0,1-yf(x)) \)

A Premissa da Similaridade

A ideia base: "Diga-me quem são seus vizinhos e eu direi quem você é."

Se um ponto desconhecido está cercado por exemplos da classe A, a hipótese mais simples é que ele também seja da classe A.

K-Nearest: Verificar mais de 1 vizinho (pode ser ruído), mas para os K mais próximos e fazemos uma votação majoritária.

Como medir a "Vizinhança"?

A distância entre o ponto P e o exemplo E em n dimensões:

d = √[ Σ (pi - ei)² ]
  • Cada característica (eixo) deve ter a mesma escala.
  • Problema: Se uma escala for 0-1 e outra 0-1000, a maior dominará o cálculo.
    • Possível solução: Z-Score ou Min-Max Scaling.
  • A "melhor" métrica de distância varia de acordo com o problema.
    • Euclidiana
    • Manhattan
    • Minkowski
    • Cosseno

Regressão Linear

Utilizada para encontrar uma relação linear entre uma variável alvo e um conjunto de atributos.

Objetivo

Modelar matematicamente a relação entre as entradas e a saída.

Hipóteses

  • A variável alvo deve ser uma combinação linear das features.
  • As variáveis explicativas devem seguir, idealmente, uma distribuição normal.

Tipos de Regressão Linear

Linear simples

Utiliza apenas uma variável explicativa.

Linear múltipla

Utiliza múltiplas variáveis.

Regressão linear simples

Encontrar os coeficientes a e b que minimizem o erro entre um alvo \(Y \in \mathbb{R}^N\) que seja linearmente dependente de um dado observacional \(X \in \mathbb{R}^{Nx1} \) independente

\(Y = aX + b\)

Regressão linear múltipla

Generalização da regressao linear simples para múltiplas variáveis \(X \in \mathbb{R}^{Nxd} \)

\(Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 ... + \beta_d X_d + \epsilon \)

Os coeficientes \( \beta \) são obtidos por algum algoritmo de otimização, como o gradiente descendente

Regressão Logística - base

  • Aplicação da regressão linear para a classificação binária
  • Utiliza uma função sigmóide (logística) para estimar a probabilidade de pertencimento a uma classe
    • Saída entre 0 e 1
    • Se o valor é 0.5, classe 1
    • Senão, classe 0
\[y = \frac{1}{1+e^{-x} \in [0,1]}\]

Regressão Logística

  • É uma forma paramétrica de estimar uma distribuição de probabilidades P(Y| X) onde \(Y \in \mathbb{R}^N\) é o espaço a estimar e \(X \in \mathbb{R}^{Nxd}\) é o espaço de atributos
  • Podem ser escritos como:
    • \[ P(Y = 1 |X) = \frac{1}{1+exp(\beta_0+\Sigma_{i=1}^{d} \beta_i X_i)}) \] e \[ P(Y = 0 |X) = \frac{exp(\beta_0+\Sigma_{i=1}^{d} \beta_i X_i}{1+exp(\beta_0+\Sigma_{i=1}^{d} \beta_i X_i)}) \]
  • Os coeficientes \(\beta_i\) são otimizados para maximizar a semelhança condicional dos dados, ou seja, a probabilidade de existência de uma Observação Y na base de dados

Regressão Logística - multiclasse

  • Estratégia "um contra todos"
    • Achar um modelo que separe bem 1 classe das demais
  • Separação de 3 classes "[green, blue, red]"
    • 1 modelo logístico para: green vs [blue+red]
    • 1 modelo logístico para: blue vs [green+red]
    • 1 modelo logístico para: red vs [blue+gren]

    Obtemos uma segmentação do espaço ao combinar os diferentes modelos

Prós e contras

  • Prós:
    • Facilidade de interpretação
    • Eficiente computacionalmente
    • Pouco sensível ao sobreajuste
    • Não necessita de normalização (transformação dos dados em pré-processamento)
  • Contras:
    • Supõe linearidade
    • Sensível ao desiquilibrio na quantidade de dados por classes
    • Performance decai a medida que os dados ficam mais complexos
    • Sensível a dados atípicos

Construção com o Gini

  • Cálculo do índice de Gini no conjunto completo:
    • \[ Gini(D) = 1 - \Sigma_{i-1}^k P_i^2 \]
  • \( P_i = \frac{Nb_{classe_i}}{Nb_{lines}} \): probabilidade de pertencer à classe i
  • O índice de Gini mede, portanto, a incerteza em um conjunto de dados:
    • Se for zero, o conjunto tem só uma única classe
    • Quanto maior for, mais diversificado o conjunto é em termos de classes

Construção com o Gini

  • Calcular o índice para o conjunto todo (successes)
  • P(sim)= 5/10
  • P(não)= ?/10
  • \(Gini(D)= 1 - (5/10)^2 - (5/10)^2\) = ?

Construção com o Gini

  • Selecionar o atributo para a divisão
  • Calcular o valor Gini ponderado para cada atributo:
  • \( Gini_{ponderado}= \Sigma_{j=1}^k \frac{N_j}{N}Gini(j) \)
    • onde \(k\) é o número de nós filhos resultante da atribuição
    • \(N_j\) é o número de amostras no nó filho
    • \(N\) é o número de amostras no nó pai
    • \(Gini(j)\) é índice do nó filho \(j\)
  • Selecione o atributo com o menor índice

    Métricas de similaridade

    • Funções de 2 paramêtros (x1,x2) que retornem um valor ℝ+ tal que:
      • D(x1,x2) = D(x2,x1) ≥ 0
      • D(x1,x2) = 0 ⟹ x1=x2
    • Métricas de distância entre 2 pontos x1 e x2:
      • Distância de Minskoswski:
        D(x1,x2)q = Σdj=1|x1,j - x2,j |q
      • Distância euclidiana ao quadrado (q=2):
        D(x1,x2)2 = Σdj=1|x1,j - x2,j |2 = (x1 - x2)T(x1 - x2)
      • Distância de Manhattan (q=1):
        D(x1,x2) = Σdj=1|x1,j - x2,j |

    Similaridade entre grupos

    • Distância D(𝒞1,𝒞2) entre 2 grupos 𝒞1 e 𝒞2:
      • Diâmetro mínimo (mais próximo do vizinho):
        Dmin(x) = min{D(xi,xj)}, xi ∈ 𝒞1, xj ∈ 𝒞2}
      • Diâmetro máximo:
        Dmax(x) = max{D(xi,xj)}, xi ∈ 𝒞1, xj ∈ 𝒞2}

    Exemplo

    • Distância D(𝒞1,𝒞2) entre 2 grupos 𝒞1 e 𝒞2:
      • Distância média:
        \[D_{média}(𝒞_1,𝒞_2) = \frac{ \sum x_i \in 𝒞_1\sum x_j \in 𝒞_2 D(x_i,x_j) }{n_{1}n_{2}}\]
      • Método de Ward:
        \[ D_{ward}(𝒞_1,𝒞_2) = \sqrt{\frac{n_1n_2}{n_1+n_2}}D(\mu_i,\mu_2) \]

    Características de um bom agrupamento

    • Minimize a inércia e maximize a distorção
    • Cada grupo 𝒞l tem como:
      • Centróide: \(\mu_l = \frac{1}{n_l} \Sigma_{i \in 𝒞_l}x_i \) onde \(n_l = card(𝒞_l)\)
      • Inércia: \(J_l = \Sigma_{i \in 𝒞_l}D^{2} (x_i,\mu_l) \)
    • Tal
    • Inércia: Distância média dentro de um grupo:
      \( J_{intra} = \Sigma J_l \) onde \( n_{𝒞} = card(𝒞) \)
    • Distorção: Distância entre grupos: \( J_{inter} = \Sigma n_l D^2(\mu_l,\mu) \) onde \( \mu = \frac{1}{N}\Sigma_i x_i \)

    • Cada ponto ou grupo é agrupado ao ponto/grupo mais próximo
    • Algoritmo:
      • inicialização:
        • Cada ponto da base de dados representa um grupo (cluter)

        • Cada grupo é adicionado à lista de grupos
        • Escolher os dois grupos \(𝒞_i\) e \(𝒞_j\) que tenham a menor distância
        • Combinar os dois grupos \(𝒞_i\) e \(𝒞_j\) em um novo \(𝒞_k\)
        • Retirar \(𝒞_i\) e \(𝒞_j\) da lista, adicionar \(𝒞_k\) na lista

    Redução de dimensionalidade

    • Seja \(X \in \mathbb{R}^{Nxd}\) uma base de dados (N pontos cada com d dimensões

      • Encontrar uma projeção de \(X \in \mathbb{Z}\) dado q < d

    Idéia central

    • Projetar dados \({x_i \in \mathbb{R}^d}$^{N}_{i=1}\) em \({z_i \in \mathbb{R}^q}$^{N}_{i=1}\) preservando ao máximo:
      • A distância entre os dados
      • A vizinhança (organização topológica)

    • Existem 2 tipos de métodos:
      • Redução linear: Análise do componente principal (PCA, em inglês)
      • Redução não-linear: Stochastic Neighbor Embedding (SNE)

    PCA - Etapa 1: Normalização dos dados

    • Normalizar os intervalos de valores de cada uma das variáveis do conjunto de dados
    • Minimizar a variância das variáveis iniciais para a continuação do algoritmo
      • \[ \{x_i \in \mathbb{R}^d\}^{N}_{i=1} \to \{x_{ij}= \frac{x_{ij}- \bar{x_j}}{\sigma_j},j=1,..,d\} \]

    PCA - Etapa 2: cálculo da matriz de covariância

    • covariância entre duas variávies \(y \in \mathbb{R}^B\) e \(y \in \mathbb{R}^N\):
        • \[ \sigma(x,y)=\sigma(y,x)=\frac{1}{N-1} \Sigma^{N}_{i=1} (x_i - \bar{x})(y_i - \bar{y}) \]
      • Matrizes de covariância de 3 variávies (x,y,z) \(\in \mathbb{R}^3 \)
      • Se \(\sigma(x,y) > 0\), quando X aumenta, y também aumenta
      • Se \(\sigma(x,y) < 0\), quando X aumenta, y diminui

    PCA - Etapa 3: cálculo dos vetores e valores próprios

    • Noção de vetores e valores próprios:
      • Conceito algébrico que permite transfomar linearmente matrizes
      • Achar vetores próprios \(X \in \mathbb{R}^d\) e seu valores próprios \(\lambda \in \mathbb{R}\) tais que \(AX = \lambda X\) dado \( A \in \mathbb{R}^{dxd}\), uma matriz quadrada
      • Resolver o sistema de equações (\(A - \lambda I)X=0\) com \(I \in \mathbb{R}^d\) a matrix identidade

    PCA - Etapa 3: cálculo dos vetores e valores próprios

    • A aplicação à matriz de covariância permite determinar as componentes principais dos dados
    • Componentes principais:
      • Construção linear das variáveis iniciais
      • As novas variáveis (componentes principais) são tão pouco correlacionadas quanto possível
      • O máximo de informações das variáveis iniciais é “comprimido” nas primeiras componentes

    t-SNE - Intuição

    • Estima a distribuição de probabilidade de vizinhança em torno de cada ponto
    • Busca minimizar as diferenças entre as distribuições de dois pontos para distribuí-los em um espaço 2D
    • O principal parâmetro do algoritmo é a perplexidade, que equivale ao número de vizinhos considerados (geralmente entre 5 e 50) pelo algoritmo
      • Uma perplexidade baixa significa que o foco está nas vizinhanças locais
      • Uma perplexidade alta significa que o foco está nas vizinhanças mais amplas
